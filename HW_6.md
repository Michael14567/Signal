# Домашнее задание 6

## Дано

Порождающая матрица:

```text
G =
[ 1 0 1 1 0 1
  1 0 1 0 1 0
  1 1 0 1 0 0 ]
```

Длина кода:

```text
n = 6
```

Так как матрица `G` имеет 3 строки, размерность кода:

```text
k = 3
```

Следовательно, код имеет `2^3 = 8` кодовых слов.

---

## 1. Кодовые слова

Кодовое слово строится по правилу:

```text
c = uG mod 2,
u ∈ F_2^3.
```

Все кодовые слова:

| Информационное слово `u` | Кодовое слово `c` | Вес `w(c)` |
|---:|:---:|---:|
| `000` | `000000` | 0 |
| `100` | `101101` | 4 |
| `010` | `101010` | 3 |
| `001` | `110100` | 3 |
| `110` | `000111` | 3 |
| `101` | `011001` | 3 |
| `011` | `011110` | 4 |
| `111` | `110011` | 4 |

Минимальный ненулевой вес равен `3`, поэтому:

```text
d_min = 3.
```

Значит, код имеет параметры:

```text
[6, 3, 3].
```

Количество исправляемых ошибок:

```text
t = floor((d_min - 1) / 2)
  = floor((3 - 1) / 2)
  = 1.
```

## 2. Построение проверочной матрицы

Проверочная матрица `H` должна удовлетворять условию:

```text
G * H^T = 0.
```

Пусть:

```text
x = (x1, x2, x3, x4, x5, x6).
```

Тогда из условия ортогональности строкам `G` получаем систему:

```text
x1 + x3 + x4 + x6 = 0
x1 + x3 + x5      = 0
x1 + x2 + x4      = 0
```

Из неё:

```text
x1 = x3 + x5
x2 = x4 + x3 + x5
x6 = x4 + x5
```

Свободные переменные:

```text
x3, x4, x5.
```

Общий вид решения:

```text
x = (x3 + x5, x3 + x4 + x5, x3, x4, x5, x4 + x5).
```

Берём три базисных решения.

| `x3` | `x4` | `x5` | `x` |
|---:|---:|---:|:---:|
| 1 | 0 | 0 | `111000` |
| 0 | 1 | 0 | `010101` |
| 0 | 0 | 1 | `110011` |

Следовательно, проверочная матрица:

```text
H =
[ 1 1 1 0 0 0
  0 1 0 1 0 1
  1 1 0 0 1 1 ]
```

Проверка:

```text
G * H^T = 0.
```

## 3. Столбцы проверочной матрицы

Столбцы `H` удобно выписать отдельно:

| Номер позиции | Столбец `h_i` |
|---:|:---:|
| 1 | `101` |
| 2 | `111` |
| 3 | `100` |
| 4 | `010` |
| 5 | `001` |
| 6 | `011` |

Если при передаче произошла одиночная ошибка в позиции `i`, то синдром равен соответствующему столбцу `h_i`.


Для одиночных ошибок:

| Ошибка `e` | Позиция ошибки | Синдром `s` |
|:---:|---:|:---:|

| `010000` | 2 | `111` |
| `001000` | 3 | `100` |
| `000100` | 4 | `010` |
| `000010` | 5 | `001` |
| `000001` | 6 | `011` |

Так как все столбцы различны и ненулевые, код исправляет одну ошибку.

# 5. Решётка по порождающей матрице

Решётка строится по кодовым словам, полученным из порождающей матрицы `G`.

Количество состояний по ярусам:

| Ярус | Число состояний |
|---:|---:|

# 8. Сравнение решёток

Решётка, построенная по порождающей матрице, и синдромная решётка, построенная по проверочной матрице, совпадают с точностью до нумерации узлов.

То есть они описывают один и тот же код, но состояния в них подписаны по-разному:

- в решётке по порождающей матрице состояния нумеруются как классы продолжений;
- в синдромной решётке состояния подписываются частичными синдромами.
