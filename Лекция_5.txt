КОНСПЕКТ ЛЕКЦИИ
Дата: 12.03.2026
Тема: продолжение построения минимальной решётки. Минимальная спэновая форма порождающей матрицы

Примечание: исходный материал рукописный. Конспект оформлен в связном виде: сохранены основные обозначения, формулы, смысл таблиц и рисунков. Часть числовых таблиц и схем передана не как буквальная перерисовка, а как объяснение того, как ими пользоваться.

1. Продолжение построения решётки

Лекция продолжает тему решёточного представления линейного кода.

Решётка строится по ярусам. Каждый ярус соответствует некоторой позиции кодового слова. Полный путь от начального узла до конечного узла задаёт одно кодовое слово.

На рисунке в начале лекции показана уже построенная решётка. Рёбра имеют двоичные метки 0 или 1. Если пройти от начального узла до конечного, то последовательность меток на рёбрах даёт кодовое слово.

В таблице под рисунком продолжается построение решётки по префиксам и суффиксам кодовых слов. Для каждого шага указываются:

    1) текущий префикс;
    2) множество возможных продолжений, то есть суффиксов;
    3) номер текущего состояния;
    4) номер следующего состояния;
    5) метка перехода, обычно 0 или 1.

Смысл таблицы такой: строка таблицы описывает одну дугу решётки. Если на данном ярусе мы находимся в некотором узле, то по метке ребра переходим в узел следующего яруса.

Например, на одном из шагов встречаются префиксы вида

    000, 101, 110, 011,

и для каждого из них выписываются допустимые суффиксы. Если два префикса имеют одинаковое множество будущих продолжений, то они могут соответствовать одному и тому же состоянию решётки.

2. Профиль минимальной решётки

Для построенной решётки указан профиль:

    z = (1, 2, 4, 4, 4, 2, 1).

Здесь z_i - число узлов на i-м ярусе.

Интерпретация профиля:

    z_0 = 1  - начальный ярус содержит один узел;
    z_1 = 2  - на первом промежуточном ярусе два узла;
    z_2 = 4;
    z_3 = 4;
    z_4 = 4;
    z_5 = 2;
    z_6 = 1  - конечный ярус снова содержит один узел.

Профиль показывает сложность решётки. Чем меньше числа z_i, тем меньше состояний нужно обрабатывать алгоритму Витерби.

3. Признак минимальности решётки

В конспекте записан следующий критерий минимальности:

    на каждом ярусе все пути, имеющие общее будущее или общее прошлое,
    проходят через один узел.

Смысл:

    если две частичные траектории дальше ведут к одним и тем же возможным продолжениям,
    то в минимальной решётке их не надо держать как разные состояния;

    если две траектории имеют одинаковую существенную информацию о прошлом,
    то они также должны быть объединены.

Иначе говоря, минимальная решётка не дублирует состояния, которые с точки зрения дальнейшего декодирования неразличимы.

Поэтому минимальная решётка кода определяется с точностью до перенумерации узлов: сами номера состояний могут быть другими, но структура переходов остаётся той же.

4. Построение решётки кода по порождающей матрице

Далее рассматривается способ построения решётки не по полному списку кодовых слов, а по порождающей матрице.

Пусть дана строка

    x = (x_1, x_2, ..., x_n).

Для неё вводятся два важных числа.

Начало строки:

    b(x) = номер позиции первого ненулевого элемента.

Конец строки:

    e(x) = номер позиции последнего ненулевого элемента.

То есть:

    b(x) = min { i : x_i != 0 },
    e(x) = max { i : x_i != 0 }.

Если строка нулевая, эти величины обычно отдельно не рассматриваются, потому что такая строка не несёт нового кодового ограничения.

5. Активные позиции строки

Активными элементами строки называются элементы, находящиеся на позициях

    b(x), b(x)+1, ..., e(x)-1.

То есть строка считается активной от своего первого ненулевого элемента до позиции перед последним ненулевым элементом.

В конспекте приведён пример строки длины 10:

    x = (0, 0, 1, 0, 1, 1, 1, 0, 0, 0).

Для неё:

    b(x) = 3,
    e(x) = 7.

Активные позиции:

    3, 4, 5, 6.

Число активных элементов в этом примере равно

    7 - 3 = 4.

Это число важно, потому что именно активные строки порождающей матрицы определяют количество состояний решётки на данном ярусе.

6. Минимальная спэновая форма порождающей матрицы

Порождающую матрицу удобно привести к минимальной спэновой форме.

Спэн строки - это отрезок от первого до последнего ненулевого элемента строки:

    [b(x), e(x)].

Минимальная спэновая форма, или МСФ, имеет следующий признак:

    все начала строк b(g_i) различны;
    все концы строк e(g_i) различны.

Здесь g_i - строки порождающей матрицы.

Идея такая: строки матрицы преобразуются элементарными операциями над GF(2), то есть сложением строк по модулю 2, чтобы сделать их интервалы активности как можно более удобными и независимыми.

7. Приведение матрицы к минимальной спэновой форме

В конспекте показан пример приведения порождающей матрицы G к МСФ.

Используются операции вида

    g_i := g_i + g_j,

где сложение выполняется по модулю 2.

Например, в записи встречаются преобразования:

    g_2 := g_2 + g_3,
    g_3 := g_3 + g_1,
    g_1 := g_1 + g_2,
    g_2 := g_2 + g_3.

Такие операции не меняют сам линейный код, потому что они только заменяют один базис пространства строк другим базисом. Код, порождённый матрицей, остаётся тем же.

Цель преобразований:

    уменьшить пересечения активных интервалов;
    добиться различия начал строк;
    добиться различия концов строк;
    получить форму, по которой проще построить решётку.

8. Активные строки на ярусе

Пусть порождающая матрица имеет строки

    g_1, g_2, ..., g_k.

На i-м ярусе активными считаются те строки, для которых

    b(g_j) <= i < e(g_j).

То есть строка g_j активна на ярусе i, если её интервал спэна проходит через этот ярус.

Обозначим число активных строк на ярусе i через

    a_i.

Тогда число состояний на этом ярусе равно

    z_i = 2^{a_i}.

Почему появляется степень двойки: для каждой активной строки можно взять коэффициент 0 или 1 в линейной комбинации. Если активных строк a_i, то возможных наборов коэффициентов 2^{a_i}.

9. Нумерация узлов решётки

Узлы на i-м ярусе нумеруются значениями вектора коэффициентов при активных строках.

Если на ярусе i активны, например, строки

    g_{j_1}, g_{j_2}, ..., g_{j_a},

то состояние можно записывать как двоичный вектор

    (u_{j_1}, u_{j_2}, ..., u_{j_a}),

где каждый коэффициент u_{j_s} равен 0 или 1.

В конспекте сформулировано так:

    узлы на ярусе i решётки нумеруются значениями вектора символов,
    соответствующего строкам, активным в i-й позиции.

Порядок перебора обычно лексикографический:

    00, 01, 10, 11,

если активны две строки, или

    000, 001, 010, 011, 100, 101, 110, 111,

если активны три строки.

10. Построение переходов между ярусами

Переход между ярусами i и i+1 строится следующим образом.

На ярусе i состояние задаёт набор коэффициентов активных строк. При переходе к следующей позиции часть строк может перестать быть активной, а часть новых строк может стать активной.

Поэтому при переходе:

    1) сохраняются коэффициенты строк, активных и до, и после перехода;
    2) удаляются коэффициенты строк, чей спэн закончился;
    3) добавляются возможные коэффициенты строк, чей спэн начался;
    4) на ребро ставится метка, равная соответствующему символу кодового слова.

Метка ребра вычисляется как сумма по модулю 2 вкладов активных строк в текущую координату.

Если текущая координата кодового слова равна c_i, то для выбранных коэффициентов u_j:

    c_i = sum u_j g_{j,i}  (mod 2),

где суммирование идёт по строкам, влияющим на данную координату.

11. Таблица комбинаций активных элементов

На странице с рисунком решётки приведена таблица комбинаций активных элементов, соответствующих состояниям порождающей матрицы.

В таблице перечисляются возможные наборы коэффициентов активных строк. Например, если активны две строки, то возможные состояния:

    00, 01, 10, 11.

Если активны три строки, то возможные состояния:

    000, 001, 010, 011, 100, 101, 110, 111.

Эти наборы подписываются у узлов соответствующего яруса.

Таким образом, решётка строится не произвольно: каждый узел имеет смысл как набор коэффициентов при активных строках матрицы.

12. Обычная решётка и секционная решётка

В конспекте показаны две схемы:

    1) обычная решётка;
    2) секционная решётка.

Обычная решётка имеет ярусы для каждой координаты кодового слова. На каждом шаге ребро обычно несёт одну двоичную метку.

Секционная решётка получается, когда несколько соседних переходов объединяются в один участок, или секцию.

В секционной решётке ребро может иметь не одну метку, а блок меток, например:

    0,0;
    1,1;
    0,1;
    1,0.

Смысл секционной решётки:

    вместо пошагового перехода по одному символу можно переходить по группам символов;
    при этом путь по решётке всё равно задаёт кодовое слово;
    секции позволяют компактнее изображать и анализировать структуру кода.

13. Связь с алгоритмом Витерби

Решётка, построенная по порождающей матрице, используется для декодирования.

Если принято слово y, то каждому ребру приписывается метрика:

    для жёсткого решения - расстояние Хэмминга между меткой ребра и соответствующим фрагментом y;
    для мягкого решения - логарифмическая метрика, связанная с правдоподобием.

Алгоритм Витерби проходит по решётке слева направо и на каждом ярусе оставляет для каждого состояния только лучший путь, пришедший в это состояние.

Именно поэтому важно иметь минимальную решётку: чем меньше состояний, тем меньше вычислений.

14. Теорема о минимальности решётки из минимальной спэновой формы

В конце лекции записана теорема.

Смысл теоремы:

    решётка, полученная из порождающей матрицы, приведённой к минимальной спэновой форме,
    является минимальной решёткой данного кода.

Иначе говоря, если матрица находится в МСФ, то построенная по активным строкам решётка уже не содержит лишних узлов, которые можно было бы склеить без потери информации о коде.

15. Идея доказательства теоремы

Доказательство опирается на критерий минимальности решётки.

На каждом ярусе состояние хранит только ту информацию, которая действительно нужна для продолжения пути:

    коэффициенты строк, активных на данном ярусе.

Если строка уже перестала быть активной, её вклад полностью относится к прошлому. Она больше не влияет на будущие координаты.

Если строка ещё не стала активной, она пока не влияет на прошлые координаты.

Если строка активна, её коэффициент нужен, потому что она одновременно связывает уже пройденную часть пути и будущую часть пути.

Поэтому узел на ярусе содержит ровно ту информацию, которая необходима, чтобы согласовать прошлое и будущее.

Если два пути имеют одинаковые активные коэффициенты на ярусе, то они имеют одинаковые допустимые продолжения. Поэтому их можно считать одним состоянием.

Если же активные коэффициенты различаются, то будущие или прошлые фрагменты будут различаться, и такие пути нельзя склеить без потери информации.

Отсюда следует минимальность решётки.

16. Основная логика построения решётки по МСФ

Алгоритм можно записать так.

Шаг 1. Взять порождающую матрицу G.

Шаг 2. Элементарными преобразованиями строк над GF(2) привести G к минимальной спэновой форме.

Шаг 3. Для каждой строки g_j найти начало и конец:

    b(g_j), e(g_j).

Шаг 4. Для каждого яруса i определить множество активных строк:

    A_i = { j : b(g_j) <= i < e(g_j) }.

Шаг 5. Построить состояния яруса i как все двоичные наборы коэффициентов активных строк:

    V_i = GF(2)^{|A_i|}.

Шаг 6. Между соседними ярусами провести рёбра, согласующие коэффициенты общих активных строк.

Шаг 7. На каждое ребро поставить метку, равную соответствующему символу кодового слова.

Шаг 8. Полученная решётка задаёт все кодовые слова как пути от начального узла к конечному.

17. Итог лекции

Главные идеи лекции:

1. Минимальная решётка не должна различать пути, у которых одинаковое существенное прошлое или одинаковое существенное будущее.

2. Профиль решётки показывает число узлов на каждом ярусе. В рассматриваемом примере профиль имеет вид

       (1, 2, 4, 4, 4, 2, 1).

3. Для построения решётки по порождающей матрице вводятся начало b(x) и конец e(x) строки.

4. Активные позиции строки - это позиции от b(x) до e(x)-1.

5. Порождающую матрицу приводят к минимальной спэновой форме, где начала строк различны и концы строк различны.

6. На каждом ярусе активные строки определяют состояния решётки.

7. Если на ярусе активно a_i строк, то число состояний равно

       z_i = 2^{a_i}.

8. Узел решётки можно понимать как набор коэффициентов при активных строках.

9. Переходы между узлами соответствуют изменению множества активных строк и несут двоичные метки кодового слова.

10. Секционная решётка объединяет несколько обычных переходов в один участок и может иметь блочные метки.

11. Решётка, построенная по порождающей матрице в минимальной спэновой форме, является минимальной для кода.

12. Такая решётка удобна для декодирования алгоритмом Витерби, потому что уменьшает число состояний и переходов.
