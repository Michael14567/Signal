Тема: декодирование по максимуму правдоподобия. Алгоритм Витерби и решётки

Примечание: исходный материал рукописный, поэтому конспект оформлен в связном виде с сохранением основных обозначений, формул и смысла лекции. Отдельные фрагменты таблиц на последних страницах были слабо различимы, поэтому они переданы как идея построения.

1. Декодирование по максимуму правдоподобия

Рассматривается дискретный канал связи, задаваемый переходными вероятностями

    P(y | x),  x in X,  y in Y.

Пусть имеется код

    C = {c_i, i = 1, ..., M} subset X^n,

где M - число кодовых слов, n - длина кодового слова.

Принятое слово обозначается

    y = (y_1, y_2, ..., y_n).

Задача декодера: по принятому слову y выбрать такое кодовое слово c_m, которое наиболее вероятно было передано.

Идея максимального правдоподобия:

    выбрать c_m, для которого P(y | c_m) максимально.

То есть декодер выбирает

    c_hat = arg max_{c_m in C} P(y | c_m).

2. Функция правдоподобия

Для каждого кодового слова c_i рассматривается функция правдоподобия

    P(y | c_i),  i = 1, ..., M.

Если канал без памяти, то вероятность принятого слова раскладывается в произведение по координатам:

    P(y | c_i) = product_{j=1}^{n} P(y_j | c_{ij}).

Это означает, что вероятность получить всё слово y равна произведению вероятностей получить каждый символ y_j при условии, что был передан символ c_{ij}.

3. Логарифм функции правдоподобия

Так как произведения вероятностей неудобны, переходят к логарифму:

    Lambda_m(y) = ln P(y | c_m).

Для канала без памяти:

    Lambda_m(y) = ln product_{j=1}^{n} P(y_j | c_{mj})
                = sum_{j=1}^{n} ln P(y_j | c_{mj}).

Максимизация P(y | c_m) эквивалентна максимизации Lambda_m(y), потому что логарифм - монотонно возрастающая функция.

Поэтому правило максимального правдоподобия можно записать как

    c_hat = arg max_{c_m in C} Lambda_m(y).

4. Отношение правдоподобия

Для двоичного случая удобно использовать логарифмическое отношение правдоподобия:

    L(y) = ln( P(y | 1) / P(y | 0) ).

Интерпретация:

    L(y) > 0  - принятое значение больше похоже на 1;
    L(y) < 0  - принятое значение больше похоже на 0;
    L(y) = 0  - оба варианта равновероятны.

Для всего принятого слова вводится вектор

    L(y) = (L(y_1), L(y_2), ..., L(y_n)).

5. Переход от двоичных символов к знаковому представлению

Для кодового слова c_m = (c_{m1}, ..., c_{mn}) вводится вектор

    xi_m = (xi_{m1}, xi_{m2}, ..., xi_{mn}),

где

    xi_{mi} = 2 c_{mi} - 1.

То есть

    c_{mi} = 0  ->  xi_{mi} = -1,
    c_{mi} = 1  ->  xi_{mi} =  1.

Такое представление удобно, потому что двоичное кодовое слово превращается в вектор из чисел -1 и 1.

6. Теорема о декодировании по максимуму правдоподобия

Для двоичного канала декодирование по максимуму правдоподобия можно свести к вычислению скалярного произведения

    L_m(y) = (xi_m, L(y)).

Здесь

    (xi_m, L(y)) = sum_{j=1}^{n} xi_{mj} L(y_j).

Теорема утверждает:

    декодирование по максимуму правдоподобия эквивалентно выбору такого кодового слова c_m,
    для которого величина L_m(y) максимальна.

Иначе говоря,

    c_hat = arg max_{c_m in C} (xi_m, L(y)).

Смысл: если координата принятого слова хорошо согласуется с координатой кодового слова, то соответствующий вклад в скалярное произведение положителен; если не согласуется - отрицателен.

7. Связь с разностью логарифмов правдоподобия

Пусть сравниваются два кодовых слова c_m и c_{m'}.

Если

    Lambda_m(y) >= Lambda_{m'}(y),

то должно выполняться

    L_m(y) >= L_{m'}(y).

Разность величин L_m(y) и L_{m'}(y) выражается через те координаты, где кодовые слова различаются.

Если в некоторой позиции одно слово имеет 1, а другое 0, вклад содержит выражение

    L(y_j) = ln( P(y_j | 1) / P(y_j | 0) ).

В итоге сравнение двух кодовых слов по максимуму правдоподобия сводится к сравнению соответствующих скалярных произведений с вектором логарифмических отношений правдоподобия.

8. Двоичный симметричный канал

Для двоичного симметричного канала вероятность ошибки обозначим p_0.

Тогда

    P(1 | 1) = 1 - p_0,
    P(0 | 1) = p_0,
    P(0 | 0) = 1 - p_0,
    P(1 | 0) = p_0.

Логарифмическое отношение правдоподобия имеет вид

    L(y) = ln( P(y | 1) / P(y | 0) ).

Если y = 1, то

    L(1) = ln( (1 - p_0) / p_0 ).

Если y = 0, то

    L(0) = ln( p_0 / (1 - p_0) ).

Обозначим

    delta = ln( (1 - p_0) / p_0 ).

Тогда при p_0 < 1/2 имеем delta > 0.

9. Максимум правдоподобия и расстояние Хэмминга

Для двоичного симметричного канала скалярное произведение можно связать с расстоянием Хэмминга:

    (xi_m, L(y)) = delta * (n - 2 d(y, c_m)),

где d(y, c_m) - расстояние Хэмминга между принятым словом y и кодовым словом c_m.

Если p_0 < 1/2, то delta > 0. Поэтому максимум величины

    delta * (n - 2 d(y, c_m))

достигается тогда, когда d(y, c_m) минимально.

Итог:

    для двоичного симметричного канала декодирование по максимуму правдоподобия
    совпадает с декодированием по ближайшему кодовому слову в метрике Хэмминга.

То есть нужно выбрать кодовое слово, ближайшее к принятому слову.

10. Поиск кратчайшего пути в решётке

Следующая часть лекции посвящена поиску кратчайшего пути в решётке и алгоритму Витерби.

Решётка - это ориентированный граф, разбитый на уровни. Каждый уровень соответствует определённому шагу или позиции в кодовом слове.

На рисунке в конспекте уровни обозначены как

    0, 1, 2, 3, 4.

Путь начинается в начальной вершине, проходит через промежуточные уровни и заканчивается в конечной вершине.

Рёбра соединяют вершины соседних уровней. Каждому ребру приписывается метрика, или вес.

11. Свойства решётки

Основные свойства решётки:

1. Вершины разбиты на уровни.
2. Нулевой и последний уровни содержат по одной вершине.
3. Решётка является направленным графом.
4. Рёбрам приписаны метрики, то есть веса.
5. Длина пути равна сумме весов рёбер, входящих в этот путь:

       length(path) = sum weights of edges in path.

Задача декодирования может быть сведена к задаче поиска пути с минимальной суммарной метрикой.

12. Алгоритм Витерби

Алгоритм Витерби используется для поиска оптимального пути в решётке.

Основная идея:

    если два пути приходят в одну и ту же вершину, то дальше имеет смысл сохранять только лучший из них.

Лучший путь называется выжившим путём.

Стандартная схема алгоритма:

1. Инициализировать метрику начальной вершины нулём.
2. Для всех остальных вершин начальные метрики считать бесконечными.
3. Последовательно переходить от уровня к уровню.
4. Для каждой вершины нового уровня рассмотреть все входящие в неё рёбра.
5. Для каждого входящего ребра вычислить кандидатную метрику:

       новая метрика = метрика предыдущей вершины + вес ребра.

6. Оставить минимальную кандидатную метрику.
7. Запомнить предшественника, через которого эта метрика достигнута.
8. После последнего уровня восстановить оптимальный путь обратным проходом.

Алгоритм Витерби не перебирает все пути полностью, а на каждом уровне отбрасывает заведомо худшие варианты.

13. Пример решётки

В конспекте приведён пример решётки с переходами между уровнями:

    уровень 0: начальная вершина;
    уровень 1: две вершины;
    уровень 2: несколько промежуточных вершин;
    уровень 3: несколько вершин перед концом;
    уровень 4: конечная вершина.

На рёбрах стоят двоичные метки 0 и 1. Эти метки показывают, какой символ соответствует данному переходу.

Каждый полный путь от начальной вершины до конечной задаёт кодовое слово. Поэтому декодирование можно понимать как выбор наиболее подходящего пути по решётке.

14. Сложность решётки и профиль

Чем меньше число вершин и рёбер в решётке, тем эффективнее декодирование.

Сложность решётки определяется её профилем:

    z = (z_0, z_1, ..., z_n),

где z_i - число вершин на i-м уровне.

Например, в конспекте встречается профиль вида

    z = (1, 2, 2, 1)

для простой решётки.

Меньший профиль означает меньшую вычислительную сложность алгоритма Витерби.

15. Построение решётки для кода

Пусть дан код

    C = {c_m, m = 1, ..., M}

длины n, где

    c_m = (c_{m1}, c_{m2}, ..., c_{mn}).

Идея построения решётки состоит в том, чтобы на каждом шаге делить кодовое слово на две части:

    c_m = (c_m^p, c_m^f),

где

    c_m^p - past, то есть уже пройденная часть слова;
    c_m^f - future, то есть оставшаяся часть слова.

Для каждого префикса c^p рассматривается множество возможных продолжений:

    F_i(c^p) = { c^f : (c^p, c^f) принадлежит C }.

Если два разных префикса имеют одинаковое множество возможных продолжений, то их можно объединить в одну вершину решётки.

Это уменьшает число вершин и делает решётку более компактной.

16. Эквивалентность состояний

На i-м уровне вершина может рассматриваться как состояние, определяемое возможными продолжениями.

Два префикса c^p и c'^p эквивалентны на уровне i, если

    F_i(c^p) = F_i(c'^p).

Тогда они ведут к одинаковым наборам будущих фрагментов, и для дальнейшего декодирования их можно не различать.

Именно это лежит в основе построения минимальной или близкой к минимальной решётки.

17. Пример линейного кода

В конспекте рассматривается пример кода, заданного порождающей матрицей

    G = [1 0 1 1 0 1]
        [1 0 1 0 1 0]
        [1 1 0 1 0 0]

Матрица имеет 3 строки, поэтому код содержит

    2^3 = 8

кодовых слов.

Они получаются как все линейные комбинации строк матрицы G над GF(2):

    C_0 = 000000,
    C_1 = 101101,
    C_2 = 101010,
    C_3 = 110100,
    C_4 = 000111,
    C_5 = 011001,
    C_6 = 011110,
    C_7 = 110011.

Порядок перечисления кодовых слов может отличаться, но множество кодовых слов определяется именно линейной оболочкой строк G.

18. Таблица разбиения по префиксам и суффиксам

Для построения решётки кодовые слова разбиваются по позициям.

На шаге i рассматриваются:

    {c^p}      - множество возможных префиксов длины i;
    F_i(c^p)  - множество возможных суффиксов после данного префикса;
    V_i        - множество вершин i-го уровня;
    z_i        - число вершин i-го уровня.

Для i = 0 имеется пустой префикс:

    c^p = empty,

и множество возможных продолжений равно всему коду:

    F_0(empty) = C.

Для i = 1 префиксы имеют длину 1:

    0 или 1.

Для i = 2 префиксы имеют длину 2:

    00, 01, 10, 11.

Для каждого такого префикса выписываются допустимые будущие части кодовых слов. Затем одинаковые множества будущих частей объединяются.

19. Связь решётки с декодированием

Каждый путь в решётке соответствует некоторому кодовому слову.

Если принято слово y, то рёбрам можно приписать метрики, измеряющие несоответствие принятому символу или фрагменту.

Например, для жёсткого решения в двоичном канале можно использовать метрику Хэмминга:

    вес ребра = 0, если символ совпадает с принятым;
    вес ребра = 1, если символ отличается от принятого.

Тогда кратчайший путь в решётке соответствует кодовому слову с минимальным расстоянием Хэмминга до принятого слова.

Для мягкого решения вместо расстояния Хэмминга используются логарифмические метрики, связанные с правдоподобием.

20. Главная идея лекции

Лекция связывает три подхода к декодированию:

1. Декодирование по максимуму правдоподобия:

       выбирается кодовое слово c_m, максимизирующее P(y | c_m).

2. Декодирование через логарифмические отношения правдоподобия:

       выбирается кодовое слово, максимизирующее (xi_m, L(y)).

3. Декодирование через решётку и алгоритм Витерби:

       выбирается путь минимальной метрики в ориентированной решётке.

Для двоичного симметричного канала эти подходы приводят к декодированию по ближайшему кодовому слову в метрике Хэмминга.

21. Итог

Основные выводы:

1. Максимум правдоподобия даёт естественное правило выбора наиболее вероятного кодового слова.
2. Для канала без памяти функция правдоподобия раскладывается в произведение по координатам.
3. Логарифм переводит произведение вероятностей в сумму.
4. В двоичном случае удобно использовать логарифмическое отношение правдоподобия L(y).
5. Декодирование можно свести к максимизации скалярного произведения (xi_m, L(y)).
6. Для двоичного симметричного канала максимум правдоподобия совпадает с минимумом расстояния Хэмминга.
7. Решётка позволяет представить кодовые слова как пути в графе.
8. Алгоритм Витерби эффективно ищет оптимальный путь, не перебирая все пути полностью.
9. Сложность решётки определяется числом вершин на уровнях, то есть её профилем.
10. Построение решётки основано на разбиении кодовых слов на прошлую и будущую части и объединении эквивалентных состояний.
