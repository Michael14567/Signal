КОНСПЕКТ ЛЕКЦИИ
Тема: линейное резервирование и декодирование линейных кодов

1. Линейные блочные коды

Рассматривается линейный (n, k)-код над полем GF(2) или, более общо, над GF(q).

n - длина кодового слова;
k - число информационных символов;
n - k - число проверочных символов;
R = k/n - скорость кода.

Для линейного кода используются две основные матрицы:

G - порождающая матрица размера k x n;
H - проверочная матрица размера (n - k) x n.

Они связаны условием ортогональности:

G H^T = 0.

Если информационное слово обозначить через u, то кодовое слово c получается как

c = uG.

Для любого кодового слова c выполняется проверочное соотношение:

c H^T = 0.

Это означает, что кодовые слова лежат в ядре проверочной матрицы H.

2. Канал передачи и вектор ошибки

Модель передачи:

информационное слово u -> кодер -> кодовое слово c -> канал с ошибкой -> принятое слово y -> декодер.

Ошибка в канале задаётся вектором

e = (e_1, e_2, ..., e_n).

Принятое слово имеет вид

y = c + e.

В двоичном случае сложение выполняется по mod 2. Если в некоторой позиции произошла ошибка, соответствующая координата вектора e равна 1; если ошибки нет - 0.

Задача декодера: по принятому слову y найти наиболее близкое кодовое слово c. Близость обычно понимается в смысле расстояния Хэмминга.

3. Сколько комбинаций ошибок возможно

Для слова длины n существует 2^n различных двоичных векторов ошибок:

000...000,
000...001,
...,
111...111.

Число кодовых слов в линейном (n, k)-коде над GF(2) равно 2^k.

Полный перебор всех кодовых слов и всех ошибок обычно неэффективен, поэтому используется синдромное декодирование.

4. Синдром

Синдром принятого слова y определяется как

s = y H^T.

Так как y = c + e, получаем:

s = y H^T = (c + e)H^T = cH^T + eH^T.

Но для кодового слова c выполнено cH^T = 0, поэтому

s = eH^T.

Главная идея: синдром зависит не от самого переданного кодового слова, а только от вектора ошибки.

Если s = 0, то принятое слово уже является кодовым словом или ошибка не обнаруживается данным кодом.

Если s != 0, то по синдрому можно попытаться восстановить наиболее вероятный вектор ошибки.

5. Синдромное декодирование

Алгоритм:

1. Получить принятое слово y.
2. Вычислить синдром:

   s = yH^T.

3. По таблице синдромов найти вектор ошибки e_i минимального веса, для которого

   e_i H^T = s.

4. Исправить слово:

   c_hat = y + e_i.

5. Если c_hat H^T = 0, то c_hat является кодовым словом.

Число возможных синдромов для двоичного (n, k)-кода равно

2^(n-k).

В общем q-ичном случае число синдромов равно

q^(n-k).

Синдромы разбивают всё пространство слов на смежные классы. Для каждого синдрома выбирают лидера класса - обычно ошибку минимального веса.

6. Пример: повторительный код (3, 1)

Рассмотрим код с порождающей матрицей

G = [1 1 1].

Тогда код содержит два слова:

C = {000, 111}.

Можно взять проверочную матрицу

H = [1 0 1]
    [0 1 1].

Для неё выполняется условие

G H^T = 0.

Синдромы для одиночных ошибок:

ошибка e    синдром s
000         00
100         10
010         01
001         11

Если, например, принято слово

y = 110,

то

s = yH^T = 11.

По таблице синдромов синдрому 11 соответствует ошибка

e = 001.

Исправляем:

c_hat = y + e = 110 + 001 = 111.

Получили кодовое слово 111.

7. Минимальное расстояние и исправление ошибок

Минимальное расстояние кода d_min - это минимальное расстояние Хэмминга между двумя различными кодовыми словами:

d_min = min d(c_i, c_j), где c_i != c_j.

Минимальное расстояние определяет число ошибок, которое код может гарантированно исправить.

Если код имеет минимальное расстояние d_min, то он гарантированно исправляет

t = floor((d_min - 1)/2)

ошибок.

Смысл: вокруг каждого кодового слова можно построить шар Хэмминга радиуса t. Если такие шары не пересекаются, то принятое слово однозначно относится к ближайшему кодовому слову.

8. Геометрическая интерпретация

Кодовые слова можно воспринимать как точки в пространстве всех слов длины n.

Исправление ошибок похоже на задачу упаковки шаров:

- центр каждого шара - кодовое слово;
- радиус шара - число ошибок, которое код исправляет;
- шары не должны пересекаться, иначе декодирование станет неоднозначным.

Задача построения кода с заданной кратностью исправления ошибок сводится к задаче упаковки шаров в дискретном пространстве Хэмминга.

Чем больше минимальное расстояние между кодовыми словами, тем больше ошибок можно исправить, но тем меньше кодовых слов можно разместить при фиксированной длине n.

9. Граница Хэмминга

Пусть имеется q-ичный код длины n с минимальным расстоянием

d_min >= 2t + 1.

Тогда каждый кодовый шар радиуса t содержит

V_q(n, t) = sum_{i=0}^{t} C_n^i (q - 1)^i

слов.

Так как шары вокруг кодовых слов не пересекаются, для числа кодовых слов M выполняется неравенство:

M * V_q(n, t) <= q^n.

Следовательно,

M <= q^n / sum_{i=0}^{t} C_n^i (q - 1)^i.

Это называется границей Хэмминга.

Здесь

C_n^i = n! / (i!(n-i)!).

Для линейного q-ичного (n, k)-кода M = q^k, поэтому граница Хэмминга ограничивает возможные параметры n, k и d_min.

10. Граница Варшамова - Гилберта

Граница Варшамова - Гилберта даёт достаточное условие существования линейного кода с заданными параметрами.

Если выполнено неравенство

q^(n-k) > sum_{i=0}^{d-2} C_{n-1}^i (q - 1)^i,

то существует q-ичный линейный (n, k)-код с минимальным расстоянием не меньше d.

Идея: если пространства достаточно, можно выбрать проверочную матрицу H так, чтобы малые наборы её столбцов были линейно независимыми. Это обеспечивает нужное минимальное расстояние.

11. Связь минимального расстояния с проверочной матрицей H

Для линейного кода минимальное расстояние можно исследовать через проверочную матрицу H.

Код имеет минимальное расстояние d тогда и только тогда, когда:

1. любые d - 1 столбцов матрицы H линейно независимы;
2. существуют d столбцов матрицы H, которые линейно зависимы.

Отсюда следует:

- ни один столбец H не должен быть нулевым;
- для d >= 3 никакие два столбца H не должны быть пропорциональны;
- для большего d требуется, чтобы любые d - 1 столбцов были линейно независимы.

Если H имеет r = n - k строк, то нельзя требовать линейной независимости более чем r столбцов. Поэтому возникает ограничение

d - 1 <= n - k,

или

d <= n - k + 1.

Это одно из базовых ограничений на параметры линейного кода.

12. Итог лекции

Основные идеи:

1. Линейный код задаётся порождающей матрицей G и проверочной матрицей H.
2. Проверочная матрица позволяет вычислять синдром принятого слова.
3. Синдром зависит только от ошибки: s = eH^T.
4. Синдромное декодирование заменяет полный перебор кодовых слов поиском ошибки по таблице синдромов.
5. Минимальное расстояние d_min определяет гарантированное число исправляемых ошибок:

   t = floor((d_min - 1)/2).

6. Исправление ошибок имеет геометрический смысл упаковки непересекающихся шаров Хэмминга.
7. Граница Хэмминга даёт верхнюю оценку на число кодовых слов.
8. Граница Варшамова - Гилберта даёт условие существования кодов с заданным расстоянием.
9. Минимальное расстояние линейного кода связано с линейной независимостью столбцов проверочной матрицы H.
